## 一、人格维度详解

每个角色有 10 维人格值（0.0 ~ 1.0），直接驱动角色的感知/决策/行为差异。

### 1.1 OCEAN 核心 5 维

| 维度 | 低值 (0.0-0.3) | 中值 (0.4-0.6) | 高值 (0.7-1.0) |
|------|----------------|----------------|----------------|
| **开放性** `openness` | 保守，抗拒变化，习惯固定的日常 | 偶尔接受新事物 | 好奇，主动探索，适应能力强 |
| **尽责性** `conscientiousness` | 随意，丢三落四，计划混乱 | 有基本条理 | 自律，有计划，注重细节 |
| **外向性** `extraversion` | 独处恢复能量，沉默寡言 | 选择性社交 | 主动社交，在人群中充电 |
| **宜人性** `agreeableness` | 竞争性强，不太顾及他人感受 | 分情况，看人 | 合作，体谅，优先考虑集体 |
| **神经质** `neuroticism` | 情绪稳定，不容易被外界影响 | 正常波动 | 情绪敏感，容易焦虑/恐惧 |

### 1.2 小说扩展 5 维

| 维度 | 低值 | 中值 | 高值 |
|------|------|------|------|
| **勇气** `courage` | 遇事退缩，避险优先 | 该上就上 | 敢于冒险，临危不惧 |
| **冷酷度** `ruthlessness` | 心软，下不去手 | 必要时候可以 | 为了目标不择手段 |
| **乐观** `optimism` | 悲观，总觉得要完 | 走一步看一步 | 相信会有转机 |
| **忠诚** `loyalty` | 利己优先 | 对特定人忠诚 | 可以牺牲自己 |
| **好奇心** `curiosity` | 对新事物无所谓 | 有了解意愿 | 主动探索未知 |

## 二、敏感 vs 钝感：完整行为映射

这是你特别关注的设计点。角色的**神经质（neuroticism）**维度决定了敏感度，并通过以下**12 个具体参数**影响引擎行为。

### 2.1 感知参数（Perception Params）

```jsonc
// 每个角色根据人格自动计算的感知参数
{
  "characterId": "zhangyuan",
  "neuroticism": 0.50,  // 中间值
  
  "perception": {
    "attentionBandwidth": 3,
    // 每 tick 最多注意到的细节数量
    // 敏感(0.8+) = 5, 中等(0.4-0.7) = 3, 钝感(<0.4) = 2
    
    "visionRadius": 4,
    // 感知半径（格子/抽象单位）
    // 敏感 = 5（更警觉，扫描更远）, 钝感 = 2
    
    "auditoryRange": 3,
    // 听觉敏感度
    // 敏感 = 5（听到风声/远处的碎裂声/脚步声）
    // 钝感 = 1（除非声音很大否则注意不到）
    
    "threatBias": 0.3,
    // 威胁信息偏见（0-1，越高越容易把中性信息解释为威胁）
    // 敏感 = 0.7（"什么声音？可能有危险"）
    // 钝感 = 0.1（"应该没事"）
    
    "detailFocus": 0.6,
    // 细节关注度（注意到微小的环境变化）
    // 敏感 = 0.9（"院门左边的栅栏第三根木桩松了"）
    // 钝感 = 0.2（"大门还行吧"）
    
    "socialCueSensitivity": 0.5
    // 社交线索敏感度（能注意到别人的表情/语气变化）
    // 敏感 = 0.8（"他今天说话不对劲"）
    // 钝感 = 0.2（完全没察觉）
  }
}
```

### 2.2 记忆参数（Memory Params）

```jsonc
{
  "memory": {
    "recencyDecay": 0.99,
    // 记忆衰减率（越高 = 记得越久）
    // 敏感 = 0.995（负面事件多年不忘）
    // 钝感 = 0.97（很快就不记得了）
    
    "importanceWeight": 1.0,
    // 检索时情感重要性的权重（poignancy 的影响）
    // 敏感 = 1.5（情绪性记忆更容易被唤起）
    // 钝感 = 0.5（不太会因为情绪而记住某件事）
    
    "relevanceWeight": 0.8,
    // 检索时相关性的权重
    // 敏感 = 1.2（容易联想到相关但不直接的事情）
    // 钝感 = 0.6（就事论事）
    
    "reflectionThreshold": 150.0,
    // 触发反思的累积阈值（越小越容易反思）
    // 敏感 = 80.0（一点小事就想半天）
    // 钝感 = 200.0（大事才反思）
    
    "maxMemoriesPerTick": 3
    // 每 tick 最多检索到的记忆数量
    // 敏感 = 5（容易被大量回忆淹没）
    // 钝感 = 2（不太会联想）
  }
}
```

### 2.3 生理参数（Physiology Params）

```jsonc
{
  "physiology": {
    "sleepQuality": 0.7,
    // 睡眠质量（0-1，影响疲劳恢复速度）
    // 敏感 = 0.3（失眠·夜醒3-4次） 
    // 钝感 = 0.95（倒下就睡）
    
    "stressAccumulation": 0.5,
    // 压力累积速度
    // 敏感 = 1.2（小事也形成压力）
    // 钝感 = 0.3（不太容易焦虑）
    
    "stressRecovery": 0.3,
    // 压力释放速度（放松/休息时的恢复效率）
    // 敏感 = 0.2（睡一觉也缓不过来）
    // 钝感 = 0.6（睡一觉就好了）
    
    "hungerSensitivity": 0.5,
    // 对饥饿的敏感度
    // 敏感 = 0.8（饿一顿就受不了）
    // 钝感 = 0.3（一天不吃也能扛）
    
    "coldResistance": 0.5
    // 耐寒能力
    // 高 = 0.8（体脂高/适应性强）
    // 低 = 0.3（容易失温）
  }
}
```

### 2.4 LLM 决策参数

```jsonc
{
  "llmDecision": {
    "planAdherence": 0.6,
    // 执行计划的程度（0-1）
    // 高尽责性 = 0.9（严格按照计划）
    // 低尽责性 = 0.3（随心情改变主意）
    
    "impulsiveness": 0.3,
    // 冲动决策概率
    // 高外向+低尽责 = 0.7（想到了就做）
    // 低外向+高尽责 = 0.1（三思而行）
    
    "socialInitiative": 0.4,
    // 主动社交的概率
    // 高外向 = 0.8（主动找别人说话）
    // 低外向 = 0.2（除非有事不然不开口）
    
    "riskTolerance": 0.4,
    // 风险承受力
    // 高勇气+低神经质 = 0.8（敢冒险）
    // 低勇气+高神经质 = 0.1（极度保守）
    
    "complianceToAuthority": 0.5
    // 服从权威的程度
    // 高宜人 = 0.8（听指挥）
    // 低宜人 = 0.2（自己说了算）
  }
}
```

## 五、"活性"系统的底层逻辑

你要求的"人物有属于人的活性"，在引擎中通过以下 5 个机制实现：

### 5.1 自发计划生成

每个角色的行为不是"外面有事件→做出反应"，而是**先有自己的计划**，事件打乱计划后重新调整：

```
每天早上6:00:
  1. 角色回顾昨天的记忆
  2. 结合当前状态（饥饿/疲劳/精神）
  3. 生成今日计划（3-6 项，LLM 生成）
  4. 开始执行计划

每 tick:
  1. 感知环境
  2. 检查是否有事件偏离计划
     ├─ 无偏离 → 继续执行计划（不调 LLM）
     └─ 有偏离 → 决策（调 LLM）→ 修改计划
```

这意味着角色即使没有任何事件触发，也会**主动地**去做计划中的事情——去巡逻、找吃的、修东西、跟人说话。

### 5.2 情绪内驱

bhava 引擎维护每个角色的 6 维情绪向量：

```
Valence (愉悦-不快)    ████████░░  0.65
Arousal (兴奋-平静)    ████░░░░░░  0.35
Dominance (支配-服从)  ██████░░░░  0.55
```

情绪变化**不依赖外部事件**——bhava 内建以下自然衰减机制：
- 情绪会自然回归基线（homeostasis）
- 饥饿/疲劳/寒冷会压低 Valence
- 睡眠不足会提高 Arousal（焦虑）
- 长期压力累积会改变基线（PTSD 效应）

这些情绪变化会**影响 LLM 的决策 prompt**（通过在 system prompt 中注入当前情绪状态），导致角色在情绪低落时更容易做出消极判断——而这不需要 LLM 显式"意识到"自己情绪不好。

### 5.3 记忆主动检索

角色在任何 tick 中，不光会感知当前环境，还会**被动联想起相关记忆**：

```rust
// 不仅是"检索相关的记忆"，而且是"随机浮现记忆"
fn random_memory_retrieval(memory: &AssociativeMemory, personality: &PersonalityTraits) -> Option<&Memory> {
    // 敏感角色：高 poignancy 记忆更容易自然浮现
    // 钝感角色：最近事件更容易自然浮现
    let score = match personality.neuroticism {
        n if n > 0.6 => memory.poignancy * 2.0,    // 敏感→情绪记忆
        n if n < 0.4 => memory.recency * 1.5,      // 钝感→近期记忆
        _ => memory.poignancy * 0.8 + memory.recency * 0.5,
    };
    // 随机抽样，但高分的概率更高
}
```

这导致角色有时会**莫名其妙地想起某件事**——就像人走在路上突然想起昨天的一件小事。

### 5.4 微行为（无 LLM 开销）

在 FastForward 模式下，角色虽然不做 LLM 决策，但仍然有一些**预设的微行为**：

```rust
fn micro_behavior(char: &mut Character, env: &Environment) -> Option<MicroAction> {
    // 1. 冷了会缩一下脖子（高频）
    if env.temperature < -10 {
        return Some(MicroAction::Shiver);
    }
    // 2. 饿了会摸肚子（中频）
    if char.state.hunger > 70 {
        return Some(MicroAction::TouchingStomach);
    }
    // 3. 累了会揉眼睛（中频）
    if char.state.fatigue > 70 {
        return Some(MicroAction::RubbingEyes);
    }
    // 4. 敏感角色会频繁检查周围（角色特定）
    if char.personality.neuroticism > 0.6 && rand() < 0.1 {
        return Some(MicroAction::LookAround);
    }
    // ...
}
```

这些微行为**会在 WebSocket 推送中体现**，让 Dashboard 上的角色看起来一直在"动"而不是静止的。

### 5.5 习惯系统

每个角色可以定义一些**日常习惯**（在角色卡中配置）：

```json
{
  "habits": [
    { "time": "06:00", "action": "起床", "probability": 0.9 },
    { "time": "06:30", "action": "检查门窗", "probability": 0.8 },
    { "time": "07:00", "action": "生火煮水", "probability": 0.95 },
    { "time": "12:00", "action": "吃午饭", "probability": 0.9 },
    { "time": "21:00", "action": "睡前检查院墙", "probability": 0.7 }
  ]
}
```

习惯是 LLM 决策的**默认选项**——如果 LLM 没有特别的理由打破习惯，角色就按习惯行动。这既节省 API 调用，也让角色行为有一致性和真实感。

## 六、角色关系变化规则

关系变化不是随机的，由以下规则驱动：

### 6.1 变化事件

| 事件 | 信任变化 | 熟悉度变化 | 好感变化 |
|------|---------|-----------|---------|
| 共同战斗 | +1~3 | +1~2 | +1~2 |
| 一方救了另一方 | +3~5 | +1 | +2~4 |
| 一方背叛 | -5~8 | 0 | -3~5 |
| 日常相处 | 0~+1 | +1 | 0~+1 |
| 长时间未见 | -1 | -1 | 0 |
| 争吵/冲突 | -2~3 | 0 | -1~3 |

### 6.2 衰减机制

如果两个角色**连续 N 天没有互动**，信任和熟悉度会缓慢衰减：

```rust
fn daily_relationship_decay(char_a: &mut Character, char_b: &mut Character, days_since_interaction: u64) {
    if days_since_interaction > 3 {
        // 信任每天衰减 0.5
        char_a.relationships[char_b].trust -= 0.5 * decay_rate(char_a.personality.loyalty);
        // 熟悉度衰减更快
        char_a.relationships[char_b].familiarity -= 1.0 * decay_rate(char_a.personality.loyalty);
    }
}

fn decay_rate(loyalty: f64) -> f64 {
    1.0 - loyalty * 0.5  // 忠诚的人衰减慢
}
```

### 6.3 关系标签升级

当信任和熟悉度达到阈值时，关系标签自动升级：

```jsonc
// 关系标签自动升级规则（硬编码在引擎中）
{
  "thresholds": [
    { "trust": 0, "familiarity": 0, "label": "陌生人" },
    { "trust": 2, "familiarity": 2, "label": "认识" },
    { "trust": 4, "familiarity": 5, "label": "熟人" },
    { "trust": 6, "familiarity": 7, "label": "朋友" },
    { "trust": 8, "familiarity": 9, "label": "挚友" },
    // 特殊情况
    { "trust": -3, "familiarity": 3, "label": "有敌意" },
    { "trust": -6, "familiarity": 4, "label": "敌人" }
  ]
}
```

## 七、角色卡编写指南

### 7.1 编写原则

1. **基底永远不变**
   - `corePersonality` 是整个角色不变的核心
   - `background` 是角色的过去，不在模拟中改变
   - `personalityTraits` 是固定的人格基线（情绪会波动，但人格不变）

2. **对话风格要具体**
   - 不要写"说话温柔"，而是"句尾喜欢带'吧'，不太用感叹号"
   - 不要写"话少"，而是"能用点头解决的绝不说一个字"

3. **背景要有可用信息**
   - LLM 会根据背景推断角色在特定情况下的行为
   - 背景中埋的细节越多，角色的行为越丰富

4. **弱点比技能更重要**
   - 弱点驱动冲突和成长
   - 没有弱点的角色是无趣的

### 7.2 角色卡模板

```json
{
  "schemaVersion": "1.0",
  "base": {
    "id": "角色英文ID（用于代码引用）",
    "name": "角色名",
    "alias": ["别名1", "别名2"],
    "gender": "男/女",
    "birthYear": 1999,
    "appearance": "外貌特征。具体、可视觉化的描述。2-3句。",
    "corePersonality": "核心性格。3-5句话，说清楚这个人的本质。",
    "background": "背景故事。包含：①末日前的身份 ②末日中的经历 ③关键转折点。5-10句话。",
    "speechStyle": "说话风格。包含：语气特点、常用词/句式、音量/语速。2-3句话。",
    "personalityTraits": { /* 10维人格，见上 */ },
    "initialSkills": [{ "name": "技能名", "level": 0.x, "category": "technical" }],
    "arc": "一句话概括角色的成长弧线",
    "initialRelationships": {},
    "habits": [{ "time": "06:00", "action": "起床", "probability": 0.9 }]
  }
}
```

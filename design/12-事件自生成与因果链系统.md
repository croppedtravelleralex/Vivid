## 一、架构总览

```
每 tick 事件流水线：
┌─────────────────────────────────────────────────────────────┐
│  1. Storyteller（节奏控制）                                   │
│     ├─ 恢复期管理器：大事件后强制安静，逐步恢复事件概率          │
│     ├─ 无聊检测器：检测系统是否陷入僵局                       │
│     └─ 无聊破局者：在无聊时寻找最合理的升级路径                │
├─────────────────────────────────────────────────────────────┤
│  2. Emergent Detector（涌现检测，零 LLM）                    │
│     ├─ 子系统碰撞检测：资源×关系×人格×情绪的交叉点            │
│     ├─ CK3 特质-事件矩阵：(Context, TraitCombo) → 事件       │
│     └─ SOC 沙堆压力溢出：压力累积超过阈值时释放               │
├─────────────────────────────────────────────────────────────┤
│  3. Cascade/Ripple Engine（级联传播，零 LLM）               │
│     ├─ 空间涟漪 BFS：事件在 petgraph 地点图上向外传播          │
│     ├─ 社交涟漪 BFS：事件在 petgraph 关系图上传播             │
│     ├─ 资源级联：依赖有向图上的链式反应                       │
│     └─ 跨域桥接：资源→健康→社交→情绪→环境的交叉传播          │
├─────────────────────────────────────────────────────────────┤
│  4. Event Memory（事件管理）                                 │
│     ├─ 遗忘曲线：旧事件活跃度衰减，不产生新级联               │
│     ├─ 诉讼时效：事件超过 N tick 后自动锁定                   │
│     └─ 未完成事业：未履行的承诺、未解决的事                   │
├─────────────────────────────────────────────────────────────┤
│  5. Probability Tree（概率树展开）                           │
│     ├─ 加权分支：A→B(70%)/C(20%)/D(10%)，概率随世界状态变化  │
│     ├─ 分支因子控制：max 3-5，低概率分支自动修剪              │

├─────────────────────────────────────────────────────────────┤
│  6. Narrative Filter（叙事滤波）                             │
│     ├─ 叙事价值评分：新颖性×突发性×影响范围×情感强度           │
│     ├─ 无聊过滤器：同类事件过渡压制                          │
│     └─ TOP-K 输出：每 tick 只展示最具叙事价值的 N 条          │
├─────────────────────────────────────────────────────────────┤
│  7. LLM Event Synthesis（LLM 事件合成）                      │
│     ├─ 时机由 SOC/规则决定（何时发生）                        │
│     ├─ 类型由 Ripple/CK3 决定（什么类型）                    │
│     └─ 细节由 LLM 决定（具体对话、描述、微调）                │
└─────────────────────────────────────────────────────────────┘
```

## 二、Storyteller（叙事节奏控制）

### 2.1 三层节奏模型

```
高节奏区（Detailed 模式必开）
  │  ████  ██    ██████    ██    ████████
  │  ████  ██    ██████    ██    ████████
中节奏区
  │  ████░░░░████████░░████░░████████░░░░
  │  ████░░░░████████░░████░░████████░░░░
低节奏区（FastForward 模式）
  │░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░
  └────────────────────────────────────────▶ 时间
           ↑           ↑           ↑
       恢复窗口    恢复窗口    恢复窗口
```

每个高节奏事件后强制一段恢复窗口，期间事件概率逐步回升。

### 2.2 Storyteller 核心

```rust
pub struct Storyteller {
    /// 威胁点——随时间积累，消耗来生成事件
    pub threat_points: f64,
    /// 恢复期管理器
    pub recovery: RecoveryManager,
    /// 无聊检测器
    pub boredom: BoredomDetector,
    /// 最近事件类型（用于多样性检测）
    pub recent_event_types: VecDeque<EventType>,
    /// 适应度——同类事件重复会降低其概率
    pub adaptation: HashMap<EventType, f64>,
}

impl Storyteller {
    pub fn tick(&mut self, world: &WorldState) -> Vec<GeneratedEvent> {
        let mut events = Vec::new();

        // 1. 恢复期递减
        self.recovery.tick();

        // 2. 威胁点积累
        let group_strength = world.characters.iter()
            .map(|c| c.skills.iter().map(|s| s.level).sum::<f64>())
            .sum::<f64>();
        self.threat_points += 0.1 + group_strength * 0.01;

        // 3. 恢复期→事件概率降低
        if !self.recovery.event_allowed() {
            let modifier = self.recovery.event_probability_modifier();
            if rng.gen::<f64>() > modifier {
                return events;
            }
        }

        // 4. 无聊检测→无聊破局
        if self.boredom.is_bored(world) {
            if let Some(event) = BoredomBreaker::break_boredom(world) {
                events.push(event);
                self.boredom.reset();
            }
        }

        // 5. 威胁点消耗生成事件
        if self.threat_points > 5.0 && rng.gen_bool(0.3) {
            let event = self.select_event(world);
            self.threat_points -= event.cost();
            self.recent_event_types.push_back(event.event_type.clone());
            self.recovery.trigger_recovery(&event);
            events.push(event);
        }

        events
    }

    fn select_event(&self, world: &WorldState) -> GeneratedEvent {
        // 选择事件类型，避免近期同类型
        let recent: HashSet<_> = self.recent_event_types.iter()
            .rev().take(3).cloned().collect();
        
        let mut candidates = EmergentDetector::detect_all(world);
        candidates.retain(|e| !recent.contains(&e.event_type));
        
        // 按叙事价值排序
        candidates.sort_by(|a, b| {
            NarrativeValueScorer::score(b, world)
                .partial_cmp(&NarrativeValueScorer::score(a, world))
                .unwrap()
        });
        
        candidates.into_iter().next()
            .unwrap_or_else(|| GeneratedEvent::default_minor(world))
    }
}
```

### 2.3 恢复期管理器

```rust
pub struct RecoveryManager {
    pub current: Option<RecoveryPeriod>,
}

pub struct RecoveryPeriod {
    pub reason: String,
    pub remaining_ticks: u64,
    pub max_ticks: u64,
}

impl RecoveryManager {
    pub fn trigger_recovery(&mut self, after_event: &Event) {
        let ticks = match after_event.severity {
            Severity::Critical => 48,   // 大事件后 48 tick 恢复
            Severity::Major => 24,
            Severity::Normal => 8,
            Severity::Minor => 3,
        };
        self.current = Some(RecoveryPeriod {
            reason: format!("{} 后恢复期", after_event.title),
            remaining_ticks: ticks,
            max_ticks: ticks,
        });
    }

    /// 恢复期内新事件的概率乘数（0~1，逐渐恢复）
    pub fn event_probability_modifier(&self) -> f64 {
        self.current.as_ref().map_or(1.0, |r| {
            let progress = 1.0 - r.remaining_ticks as f64 / r.max_ticks as f64;
            // ease-out: 开始恢复慢，后面加速
            1.0 - (progress * std::f64::consts::FRAC_PI_2).cos()
        })
    }

    pub fn tick(&mut self) {
        if let Some(ref mut r) = self.current {
            r.remaining_ticks = r.remaining_ticks.saturating_sub(1);
            if r.remaining_ticks == 0 {
                self.current = None;
            }
        }
    }
}
```

### 2.4 无聊检测与破局

```rust
pub struct BoredomDetector {
    pub consecutive_quiet_ticks: u64,
    pub event_density: VecDeque<(u64, u64)>,  // (tick, count)
}

impl BoredomDetector {
    pub fn is_bored(&self, world: &WorldState) -> bool {
        let mut score = 0.0;

        // 长时间无事件
        if self.consecutive_quiet_ticks > 10 { score += 0.4; }

        // 角色状态几乎没变
        let state_delta: f64 = world.characters.iter()
            .map(|c| c.state.total_delta_since_last_tick())
            .sum();
        if state_delta < 0.5 { score += 0.3; }

        // 关系图僵化
        let graph_delta = world.relationship_graph.total_edge_weight_change();
        if graph_delta < 0.05 { score += 0.3; }

        score >= 0.7
    }
}

pub struct BoredomBreaker;

impl BoredomBreaker {
    /// 不是"随机扔事件"，而是在当前状态下找最合理的升级路径
    pub fn break_boredom(world: &WorldState) -> Option<GeneratedEvent> {
        // 优先级 1：如果有未完成的社会冲突，升级它
        if let Some(event) = escalate_social_tension(world) { return Some(event); }
        
        // 优先级 2：如果资源持续消耗，触发一个短缺事件
        if let Some(event) = escalate_resource_crisis(world) { return Some(event); }
        
        // 优先级 3：如果全员留守太久，引入外部威胁
        if world.characters.iter().all(|c| c.is_at_home()) {
            return Some(GeneratedEvent {
                event_type: "external_stimulus".into(),
                description: format!("远处传来奇怪的声音"),
                severity: Severity::Minor,
                participants: vec![world.characters.first()?.id],
            });
        }

        None
    }
}
```

## 三、Emergent Detector（涌现检测引擎）

### 3.1 子系统碰撞检测

事件不是"被触发"的，而是子系统碰撞的副作用：

```rust
pub trait SubsystemCollisionDetector {
    fn detect(&self, world: &WorldState) -> Option<EmergentEvent>;
}

/// 所有碰撞检测器的集合
pub struct EmergentDetector {
    detectors: Vec<Box<dyn SubsystemCollisionDetector>>,
}

impl EmergentDetector {
    pub fn detect_all(world: &WorldState) -> Vec<EmergentEvent> {
        let mut results = Vec::new();
        for detector in &self.detectors {
            if let Some(event) = detector.detect(world) {
                results.push(event);
            }
        }
        results
    }
}

/// 示例：资源×关系碰撞——资源紧张+低信任=资源冲突
struct ResourceRelationCollision;

impl SubsystemCollisionDetector for ResourceRelationCollision {
    fn detect(&self, world: &WorldState) -> Option<EmergentEvent> {
        for location in &world.locations {
            let critical: Vec<_> = location.resources.iter()
                .filter(|(_, r)| r.current as f64 / r.max as f64 < 0.2)
                .collect();
            if critical.is_empty() { continue; }

            let occupants = world.characters_at(location.id);
            for pair in occupants.windows(2) {
                let rel = world.relationship_graph.get(pair[0], pair[1]);
                if rel.trust < 3 {
                    return Some(EmergentEvent::resource_conflict(
                        critical[0].0.clone(), pair[0], pair[1]
                    ));
                }
            }
        }
        None
    }
}
```

### 3.2 CK3 特质-事件矩阵

```rust
pub struct TraitEventMatrix {
    /// 二维矩阵：(ContextTag, TraitCombo) → Vec<(EventTemplate, BaseWeight)>
    entries: HashMap<(ContextTag, TraitCombo), Vec<WeightedEvent>>,
}

pub struct WeightedEvent {
    pub template: EventTemplate,
    pub base_weight: f64,
    pub modifiers: Vec<WeightModifier>,  // 根据世界状态动态调整权重
}

impl TraitEventMatrix {
    pub fn roll(&self, character: &Character, context: &Context) -> Vec<GeneratedEvent> {
        let mut results = Vec::new();
        for combo in character.traits.active_combinations() {
            let key = (context.tag.clone(), combo);
            if let Some(entries) = self.entries.get(&key) {
                for weighted in entries {
                    let mut weight = weighted.base_weight;
                    for modifier in &weighted.modifiers {
                        weight *= modifier.adjust(character, &world);
                    }
                    if rng.gen::<f64>() < weight {
                        results.push(weighted.template.instantiate(character, context));
                    }
                }
            }
        }
        results
    }
}
```

### 3.3 SOC 沙堆压力溢出

```rust
/// 每个角色有一个"压力沙堆"，累积到阈值后溢出为事件
pub struct CrisisSandpile {
    pub stress: HashMap<Uuid, f64>,
    pub threshold: f64,  // 默认 4.0
}

impl CrisisSandpile {
    pub fn add_stress(&mut self, char_id: Uuid, amount: f64, world: &mut WorldState) -> Vec<EmergentEvent> {
        let stress = self.stress.entry(char_id).or_insert(0.0);
        *stress += amount;

        if *stress >= self.threshold {
            *stress = 0.0;
            // 压力溢出→可能产生事件
            // 溢出效应会传播到社交邻居（级联）
            self.cascade(char_id, world)
        } else {
            vec![]
        }
    }

    fn cascade(&mut self, origin: Uuid, world: &mut WorldState) -> Vec<EmergentEvent> {
        let mut events = Vec::new();
        
        // 压力传播到社交邻居
        for neighbor in world.relationship_graph.neighbors(&origin) {
            let stress = self.stress.entry(neighbor).or_insert(0.0);
            *stress += 0.5;  // 每次传播+0.5
            if *stress >= self.threshold {
                *stress = 0.0;
                events.push(EmergentEvent::stress_cascade(neighbor, origin));
            }
        }

        events
    }
}
```

## 四、Cascade/Ripple Engine（级联传播）

### 4.1 空间涟漪（BFS on 地点图）

```rust
pub struct SpatialPropagation {
    pub max_radius_km: f64,     // 最大传播半径
    pub attenuation: f64,       // 每公里衰减
}

impl SpatialPropagation {
    pub fn propagate(&self, event: &Event, location_graph: &LocationGraph, origin: NodeIndex) -> Vec<PropagatedEvent> {
        use petgraph::algo::dijkstra;
        
        let distances = dijkstra(location_graph, origin, None, |e| e.weight().distance_km);
        
        distances.into_iter()
            .filter(|(_, d)| *d > 0.0 && *d <= self.max_radius_km)
            .filter_map(|(node, dist)| {
                let prob = (1.0 - dist / self.max_radius_km) * self.attenuation;
                if rng.gen::<f64>() > prob { return None; }
                
                Some(PropagatedEvent {
                    location: node,
                    severity_scale: 1.0 - (dist / self.max_radius_km) * 0.7,
                    delay_ticks: (dist * 2.0) as u64,
                })
            })
            .collect()
    }
}
```

### 4.2 社交涟漪（BFS on 关系图）

```rust
pub struct SocialPropagation {
    pub max_hops: usize,       // 最多传几层
    pub gossip_chance: f64,    // 每跳传播概率
}

impl SocialPropagation {
    pub fn propagate(&self, event: &SocialEvent, graph: &RelationshipGraph) -> Vec<SocialRipple> {
        let mut ripples = Vec::new();
        let start = graph.node_index(event.actor_id);

        // BFS 逐层传播
        for (node_idx, hops) in graph.bfs_with_distance(start) {
            if hops > self.max_hops { break; }
            if hops == 0 { continue; }

            let prob = self.gossip_chance / hops as f64;
            if rng.gen::<f64>() > prob { continue; }

            let reaction = graph[node_idx].personality.react_to_hearsay(event);
            ripples.push(SocialRipple {
                target: node_idx,
                change: reaction,
                delay_ticks: hops as u64,
            });
        }

        ripples
    }
}
```

### 4.3 资源级联（依赖有向图）

```rust
pub struct ResourceCascade {
    /// 资源依赖图：fuel → electricity → water → hygiene
    pub dependencies: HashMap<ResourceType, Vec<(ResourceType, f64)>>,
}

impl ResourceCascade {
    pub fn propagate(&self, change: &ResourceChange, world: &mut WorldState) -> Vec<CascadeEvent> {
        let mut queue = VecDeque::new();
        queue.push_back(change.resource_type);
        let mut events = Vec::new();

        while let Some(resource) = queue.pop_front() {
            if let Some(dependents) = self.dependencies.get(&resource) {
                for (dep, conversion) in dependents {
                    let delta = world.get_resource_delta(resource);
                    if delta.abs() > 0.15 {  // 变化>15%才传播
                        let impact = delta * conversion;
                        world.apply_resource_change(dep, impact);
                        events.push(CascadeEvent {
                            resource: dep.clone(),
                            impact,
                            severity: if impact.abs() > 0.3 { "major" } else { "minor" },
                        });
                        queue.push_back(dep);
                    }
                }
            }
        }

        events
    }
}
```

## 五、跨域桥接

### 5.1 域定义

```
Resource Domain (资源域)
  ├─ food, fuel, wood, medicine
  └─ 触达点：消耗完→健康域（饥饿↑）
  
Health Domain (健康域)
  ├─ hp, hunger, warmth, fatigue, injuries
  └─ 触达点：受伤→社交域（他人来帮忙）
  
Social Domain (社交域)
  ├─ trust, familiarity, sentiment
  └─ 触达点：冲突→资源域（争夺物资）
  
Emotion Domain (情绪域, bhava)
  ├─ valence, arousal, dominance, stress
  └─ 触达点：长期高压→健康域（免疫↓）
  
Environment Domain (环境域)
  ├─ temperature, weather, season
  └─ 触达点：暴风雪→资源域（无法外出）
```

### 5.2 域桥接函数

```rust
pub struct DomainBridge {
    pub rules: Vec<DomainRule>,
}

pub struct DomainRule {
    pub source: Domain,
    pub target: Domain,
    pub map: Box<dyn Fn(f64, &WorldState) -> (f64, f64)>,  // (概率, 幅度)
}

impl DomainBridge {
    pub fn propagate(&self, change: &DomainChange, world: &WorldState) -> Vec<DomainChange> {
        let mut results = Vec::new();
        for rule in &self.rules {
            if rule.source != change.domain { continue; }
            let (prob, magnitude) = (rule.map)(change.magnitude, world);
            if rng.gen::<f64>() < prob {
                results.push(DomainChange {
                    domain: rule.target,
                    magnitude,
                    delay_ticks: rng.gen_range(1..4),
                });
            }
        }
        results
    }
}

// 示例桥接规则
let resource_to_health = DomainRule {
    source: Domain::Resource,
    target: Domain::Health,
    map: Box::new(|food_change, _| {
        let hunger = -food_change * 2.0;
        let prob = if food_change < 0.0 { 1.0 } else { 0.3 };
        (prob, hunger)
    }),
};
```

### 5.3 域缓冲（Domain Buffer）

小波动不跨域传播，被缓冲吸收：

```rust
pub struct DomainBuffer {
    pub value: f64,
    pub threshold: f64,  // 超过此阈值才传播
    pub decay: f64,      // 自然衰减速率
}

impl DomainBuffer {
    /// 输入变化量，输出超过阈值才传播的部分
    pub fn absorb(&mut self, change: f64) -> f64 {
        self.value += change;
        if self.value.abs() >= self.threshold {
            let overflow = self.value;
            self.value = 0.0;
            overflow
        } else {
            0.0  // 被缓冲吸收
        }
    }
    
    pub fn tick_decay(&mut self) {
        self.value *= 1.0 - self.decay;
        if self.value.abs() < 0.01 { self.value = 0.0; }
    }
}
```

## 六、概率加权事件树

### 6.1 基础结构

```rust
pub struct EventBranch {
    pub description: String,
    pub base_weight: f64,
    pub modifiers: Vec<WeightModifier>,       // 动态调整因子
    pub sub_branches: Vec<EventBranch>,       // 递归子分支
}

pub struct WeightModifier {
    pub condition: Box<dyn Fn(&WorldState) -> f64>,  // 返回权重乘数
}

impl EventBranch {
    pub fn effective_weight(&self, world: &WorldState) -> f64 {
        let mut w = self.base_weight;
        for modifier in &self.modifiers {
            w *= (modifier.condition)(world);
        }
        w
    }
    
    pub fn roll(&self, world: &WorldState) -> Option<&EventBranch> {
        let total: f64 = self.sub_branches.iter()
            .map(|b| b.effective_weight(world)).sum();
        if total == 0.0 { return None; }
        
        let roll = rng.gen::<f64>() * total;
        let mut cumulative = 0.0;
        for branch in &self.sub_branches {
            cumulative += branch.effective_weight(world);
            if roll <= cumulative { return Some(branch); }
        }
        None
    }
}
```

### 6.2 分支因子控制

```rust
pub struct BranchControl {
    pub max_children: usize,            // 默认 3-5
    pub significance: f64,              // 权重低于此的剪枝
}

impl BranchControl {
    pub fn prune(&self, branches: &mut Vec<EventBranch>, world: &WorldState) {
        // 1. 低概率分支剪枝
        branches.retain(|b| b.effective_weight(world) >= self.significance);
        
        // 2. 如果超过 max_children，保留 top-N
        if branches.len() > self.max_children {
            branches.sort_by(|a, b| {
                b.effective_weight(world).partial_cmp(&a.effective_weight(world)).unwrap()
            });
            branches.truncate(self.max_children);
        }
    }
}
```

## 七、事件记忆与遗忘

### 7.1 Ebbinghaus 遗忘曲线

```rust
pub struct ForgettingCurve {
    pub half_life_ticks: u64,       // 基础半衰期
    pub importance_factor: f64,     // 重要性乘数
}

impl ForgettingCurve {
    /// 事件活跃度（0~1），低于 0.2 不产生级联
    pub fn activation(&self, event: &HistoricalEvent, current_tick: u64) -> f64 {
        let age = current_tick.saturating_sub(event.occurred_at_tick);
        if age == 0 { return 1.0; }
        
        let adjusted = self.half_life_ticks as f64 
            * (1.0 + event.poignancy * self.importance_factor);
        
        (-(age as f64) / adjusted).exp()
    }
    
    /// 仍活跃的事件（能产生级联的）
    pub fn active_events<'a>(&self, events: &'a [HistoricalEvent], tick: u64) -> impl Iterator<Item = &'a HistoricalEvent> {
        events.iter().filter(move |e| self.activation(e, tick) >= 0.2)
    }
}
```

```rust
            Severity::Major => 500,
            Severity::Normal => 200,
            Severity::Minor => 50,
```

```rust
    }

pub struct PendingPromise {
    pub id: Uuid,
    pub promiser: Uuid,
    pub promisee: Uuid,
    pub content: String,
    pub deadline_tick: Option<u64>,
    pub reminder_prob: f64,        // 每 tick 被提起的概率
    pub escalation: Option<Box<dyn Fn(&WorldState) -> GeneratedEvent>>,
}

impl UnfinishedBusiness {
    pub fn tick(&mut self, world: &WorldState) -> Vec<GeneratedEvent> {
        let mut events = Vec::new();
        
        self.pending.retain(|item| {
            // 截止日期过了 → 违约
            if let Some(deadline) = item.deadline_tick {
                if world.tick_count > deadline {
                    events.push(GeneratedEvent {
                        event_type: "promise_broken".into(),
                        description: format!("{} 未能履行对 {} 的承诺：{}",
                            world.char_name(item.promiser),
                            world.char_name(item.promisee),
                            item.content),
                    });
                    return false;
                }
            }
            
            // 概率提醒
            if rng.gen_bool(item.reminder_prob) {
                events.push(GeneratedEvent {
                    event_type: "reminder".into(),
                    description: format!("{} 想起 {} 还没兑现承诺",
                        world.char_name(item.promisee), item.content),
                });
            }
            
            true
        });
        
        events
    }
}
```

## 八、叙事价值过滤

### 8.1 评分系统

```rust
pub struct NarrativeValueScorer;

impl NarrativeValueScorer {
    pub fn score(event: &GeneratedEvent, world: &WorldState) -> f64 {
        let mut score = 0.0;
        score += novelty(event) * 2.0;          // 新颖性
        score += unexpectedness(event, world) * 1.5;  // 突发性
        score += impact_scope(event) * 1.0;     // 影响范围
        score += cascade_potential(event) * 0.8; // 连锁潜力
        score += character_importance(event, world) * 1.2; // 角色重要度
        score += emotional_impact(event) * 0.5; // 情感强度
        score
    }
}

/// 新颖性——与最近 10 个事件比较相似度
fn novelty(event: &GeneratedEvent) -> f64 {
    let recent = RECENT_EVENTS.with(|e| e.borrow().recent(10));
    let max_sim = recent.iter()
        .map(|e| event.similarity_to(e))
        .max_by(|a, b| a.partial_cmp(b).unwrap())
        .unwrap_or(0.0);
    1.0 - max_sim
}

/// 突发性——该事件在当前状态下有多"意外"
fn unexpectedness(event: &GeneratedEvent, world: &WorldState) -> f64 {
    let predicted_prob = event.predicted_probability(world);
    1.0 - predicted_prob
}

/// 连锁潜力——可能产生多少后续分支
fn cascade_potential(event: &GeneratedEvent) -> f64 {
    let cascade_count = event.possible_cascades().len() as f64;
    (cascade_count / 5.0).min(1.0)
}
```

```rust
impl BoredomFilter {
    pub fn should_suppress(&mut self, event: &GeneratedEvent) -> bool {
        self.recent_types.push_back(event.event_type.clone());
        if self.recent_types.len() > 20 { self.recent_types.pop_front(); }

        let recent_5: Vec<_> = self.recent_types.iter().rev().take(5).collect();
        let same = recent_5.iter().filter(|t| *t == &&event.event_type).count();

        same >= 3  // 最近 5 个中超过 3 个同类→压制
    }
}
```

```rust
fn validate_llm_event(llm_output: &LLMEventOutput, template: &EventTemplate, world: &WorldState) -> Result<(), ValidationError> {
    // 1. 不改变参与者
    if llm_output.participants != template.participants {
        return Err(ValidationError::ParticipantsChanged);
    }
    
    // 2. 不改变事件类型
    if llm_output.event_type != template.event_type {
        return Err(ValidationError::TypeChanged);
    }
    
    // 3. 不引入不存在的角色/地点
    for name in llm_output.mentioned_names() {
        if !world.characters.iter().any(|c| c.name == name) {
            return Err(ValidationError::HallucinatedCharacter(name));
        }
    }
    
    // 4. 不改变环境事实
    if (llm_output.temperature - world.environment.temperature).abs() > 5.0 {
        return Err(ValidationError::TemperatureMismatch);
    }
    
    Ok(())
}
```

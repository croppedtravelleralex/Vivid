# 零下家园 · 小说世界模拟引擎 — 项目总览与技术栈

> 本文档是整套设计文档的入口，定义项目定位、技术选型、目录结构和整体架构。

---

## 一、项目定位

### 是什么
一个**小说世界时间线模拟引擎**。自动推进小说世界的时间线，让角色在无人干预的情况下自发行动、感知、思考，产出可用于小说写作的素材（对话、冲突、情节转折）。

### 解决什么问题
| 问题 | 现有做法 | 本方案 |
|------|---------|--------|
| 角色一致性 | AI 靠上下文窗口记忆，很容易漂移 | 角色卡持久化 + 每次注入 system prompt |
| 多角色并行 | 手动轮流写每个角色的视角 | 引擎自动并行决策所有角色 |
| 时间线管理 | 手写"三天后""一周过去了" | 引擎精确到分钟推进，自动触发事件 |
| 世界演化 | 全凭作者脑补温度和资源变化 | 环境系统自动计算温度/资源/季节 |
| 素材产出 | 每次从零开始写 | 引擎推演→作者删改→可用的正文段落 |

### 技术定位
- 单机运行，纯本地，不需要联网（除 LLM API 调用外）
- Rust 后端模拟引擎 + Vue 3 前端 Dashboard
- 轻量：< 500MB 内存，无需 GPU
- 核心文件：`characters/*.json`（角色卡）、`world/*.json`（世界设定）→ 全部纯文本可编辑

## 二、技术栈

### 后端（Rust 模拟引擎）

| 层 | crate | 用途 |
|---|---|---|
| **Web 框架** | `axum 0.8` | 提供 REST API + WebSocket |
| **异步运行时** | `tokio 1.x` | 异步任务、并行 LLM 调用 |
| **HTTP 客户端** | `reqwest 0.12` | 调用 DeepSeek API |
| **序列化** | `serde` + `serde_json` + `bincode` | JSON API + 二进制快照 |
| **图结构** | `petgraph 0.7` | 角色关系图 + 地点拓扑图 |
| **实体存储** | `hecs 0.11` | 轻量 ECS，无游戏引擎包袱 |
| **情绪/人格** | `bhava 2.0` | 15 维人格 + 6D 情绪向量 + 昼夜节律 |
| **事件队列** | `std::collections::BinaryHeap` | 按时间排序的待办事件 |
| **嵌入式 DB** | `redb 4.1` | 持久化检查点和配置 |
| **时间** | `chrono 0.4` | 模拟时间计算 |
| **ID 生成** | `uuid 1.x` (v7) | 实体唯一 ID，时间排序 |
| **日志** | `tracing 0.1` | 结构化日志 |
| **错误处理** | `thiserror` + `anyhow` | 标准 Rust 错误处理 |
| **模板引擎** | `minijinja 2.x` | Prompt 模板渲染 |
| **速率限制** | `governor 0.10` | LLM API 调用限流 |
| **退避重试** | `tokio-retry 0.3` | LLM 调用容错 |
| **配置** | `dotenvy 0.15` | .env 配置加载 |
| **WebSocket** | `tokio-tungstenite`（通过 axum 集成） | 实时推送模拟状态 |

### 前端（Vue 3 Dashboard）

| 层 | 依赖 | 用途 |
|---|---|---|
| **框架** | Vue 3 (Composition API + `<script setup>`) | UI 框架 |
| **构建** | Vite 7 | 开发/构建 |
| **路由** | vue-router 4 | 多页面导航 |
| **HTTP** | axios | 调用 Rust 后端 API |
| **实时通信** | WebSocket (原生) | 接收模拟推演实时事件 |
| **关系图** | d3-force + d3-zoom | 力导向关系图可视化 |
| **时间线** | d3-scale + d3-axis | 时间线事件可视化 |
| **图表** | chart.js (vue-chartjs) | 环境趋势图、角色状态图 |
| **状态管理** | Vue 3 reactive/ref + provide/inject | 轻量状态管理（无需 Pinia） |
| **国际化** | vue-i18n | 多语言（中/英） |
| **UI 样式** | 纯 CSS (无 UI 框架) | 轻量、自定义 |


### 为什么选 axum 而不是 Tauri

| | Tauri | Axum |
|---|---|---|
| 通信方式 | IPC (invoke) | HTTP + WebSocket |
| 调试 | 需编译后运行 | 浏览器 DevTools 直接调试 |
| 前端框架限制 | 无（但需 webview 兼容） | 无 |
| 部署 | 打包成桌面 exe | 任意浏览器访问 |
| 多人协作 | 单机 | 可局域网多人看模拟进度 |
| 复杂度 | 需处理文件系统/窗口等 | 纯 Web 服务 |

**选 Axum**：MiroFish 也是 Web 模式（Flask + Vue），开发调试方便，部署灵活。

## 五、核心设计原则

### 1. 角色驱动，而非剧情驱动
引擎不预设剧情走向。剧情是角色在环境中按照人设自发行动产生的**涌现结果**。作者设定的是初始条件和角色性格，而非固定的情节线。

### 2. 增量更新，而非全量重写
角色卡不靠 LLM 全量重写。每次事件只做**增量补丁**：追加记忆、增减技能、更新关系值。核心性格永不改变。这解决了 AI 写长篇的角色一致性痛点。

### 3. 分层决策，按需调用 LLM
```
无事 → FastForward（零 LLM，只跑数值）
有事 → Detailed（调 LLM 做决策）
```
不在"无事可做"时浪费 API 调用。90% 的模拟时间处于 FastForward 模式。

### 4. 可回溯，不丢失
每次 tick 都是确定性的（给定相同的随机种子）。每 N tick 保存检查点，支持回退到任意历史时间点。事件日志追加写，永不删除。

### 5. 数据即文件
所有角色卡、世界设定、剧情事件都是 `.json` 文本文件。可直接用编辑器修改，改完即生效（引擎热加载）。不依赖任何数据库服务。

## 六、小说适配说明

本引擎针对"零下家园"末世纪元定制，但架构上不绑定特定小说设定。更换 `data/characters/`、`data/world/`、`data/plot/` 下的 JSON 数据即可适配其他小说世界观。

当前适配：
- 时间跨度：2025-12-03 ~ 2035+
- 温度区间：-5℃ ~ -70℃（十年持续降温）
- 核心冲突：极寒生存 + 丧尸 + 人性溃败
- 角色规模：5 个初始角色（逐步扩充）
- 地点规模：4 个初始地点（逐步扩展至 20+）

# 零下家园 · 小说世界模拟引擎 — 核心数据流与通信协议

> 定义 Rust 后端与 Vue 前端之间的所有交互协议：HTTP API 请求/响应格式、WebSocket 事件格式、错误码规范。

---

## 一、通信模式总览

```
┌────────────────────┐         HTTP REST          ┌────────────────────┐
│                    │ ◄────（前端主动请求）───── │                    │
│    Vue 3 前端       │                            │    Rust 后端       │
│                    │ ──── WebSocket 推送 ──────▶ │    (Axum)          │
│                    │    （引擎主动推送）          │                    │
└────────────────────┘                            └────────────────────┘
```

| 模式 | 方向 | 用途 | 频率 |
|------|------|------|------|
| HTTP GET | 前端 → 后端 | 首次加载、手动刷新 | 按需 |
| HTTP POST | 前端 → 后端 | 控制指令（开始/暂停/保存） | 用户操作时 |
| WebSocket 推送 | 后端 → 前端 | tick 事件、状态更新 | 每 tick |

---

## 二、API 路径与版本

所有 API 以 `/api/v1` 为前缀：

| 组 | 前缀 |
|----|------|
| 世界 | `GET /api/v1/world/*` |
| 角色 | `GET /api/v1/characters/*` |
| 关系图 | `GET /api/v1/graph/*` |
| 时间线 | `GET /api/v1/timeline/*` |
| 模拟控制 | `POST /api/v1/simulation/*` |
| 检查点 | `POST /api/v1/checkpoint/*` |
| WebSocket | `WS /api/v1/ws` |

## 四、WebSocket 协议

### 4.1 连接

```
客户端:  WS /api/v1/ws
服务端:  101 Switching Protocols
```

连接建立后，后端立即发送 `Connected` 事件：

```json
{ "type": "connected", "data": { "sessionId": "sess-xxx", "serverTime": "2025-12-08T14:00:00Z" } }
```

### 4.2 事件格式

所有 WebSocket 消息使用统一信封：

```json
{
  "type": "事件类型",
  "timestamp": "2025-12-08T14:00:05",
  "data": { ... }
}
```

### 4.3 事件类型表

| type | 频率 | data 内容 |
|------|------|-----------|
| `heartbeat` | 每 5s | `{}` |
| `tick` | 每 FastForward tick | `{ tick, time, speed }` |
| `detailed_tick` | 每 Detailed tick | `{ tick, time, decisions, events }` |
| `speed_changed` | 切换时 | `{ from, to, reason }` |
| `character_update` | 状态变化时 | `{ char_id, name, diff: { hp, hunger, warmth, fatigue, mental }, location }` |
| `relationship_changed` | 关系变化时 | `{ char_a, char_b, delta: { trust, familiarity }, new_values: { trust, familiarity, sentiment }, reason }` |
| `event_triggered` | 事件发生时 | `{ event_id, event_type, title, severity, participants, location_id }` |
| `environment_update` | 环境变化时 | `{ temperature, weather, season, daylight }` |
| `llm_call` | LLM 调用时 | `{ char_name, phase, latency_ms, tokens }` |
| `log` | 引擎内部日志 | `{ level, message, module }` |
| `council_completed` | 议会完成时 | `{ round, summary, cost }` |
| `character_added` | 角色新增时 | `{ char_id, name, location }` |
| `character_removed` | 角色移除时 | `{ char_id, name, reason }` |
| `tag_updated` | 标签变更时 | `{ thread_id, status, freshness }` |

### 4.5 前端重连策略

```javascript
// WebSocket 断开时自动重连，指数退避
const RECONNECT_DELAYS = [1000, 2000, 4000, 8000, 15000, 30000]

function handleDisconnect() {
  let attempt = 0
  function reconnect() {
    const delay = RECONNECT_DELAYS[Math.min(attempt, RECONNECT_DELAYS.length - 1)]
    console.log(`WebSocket 断开，${delay}ms 后重连 (attempt ${attempt + 1})`)
    setTimeout(() => {
      connect()
    }, delay)
    attempt++
  }
}
```

## 五、错误码规范

### 5.1 HTTP 状态码

| 状态码 | 含义 | 场景 |
|--------|------|------|
| 200 | OK | 正常返回 |
| 400 | Bad Request | 参数错误 |
| 404 | Not Found | 角色/检查点不存在 |
| 409 | Conflict | 引擎正在运行无法执行此操作 |
| 429 | Too Many Requests | LLM 限流中 |
| 500 | Internal Server Error | 引擎内部错误 |
| 503 | Service Unavailable | LLM API 不可用 |

### 5.2 错误响应格式

```json
{
  "status": "error",
  "error": "ERROR_CODE",
  "message": "人类可读的错误描述",
  "details": {}  // 可选，额外调试信息
}
```

### 5.3 错误码表

| code | HTTP | 含义 |
|------|------|------|
| `ENGINE_RUNNING` | 409 | 引擎运行中，禁止操作 |
| `ENGINE_PAUSED` | 409 | 引擎已暂停，此操作需要运行状态 |
| `CHARACTER_NOT_FOUND` | 404 | 角色 ID 不存在 |
| `CHECKPOINT_NOT_FOUND` | 404 | 检查点 tag 不存在 |
| `INVALID_SPEED` | 400 | 速度模式参数错误 |
| `LLM_TIMEOUT` | 503 | LLM API 超时 |
| `LLM_RATE_LIMITED` | 429 | LLM API 速率限制 |
| `INTERNAL_ERROR` | 500 | 引擎内部错误 |
| `SERIALIZE_ERROR` | 500 | 序列化/反序列化失败 |

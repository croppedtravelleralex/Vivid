## 二、初始关系图构建

### 2.1 从角色卡构建 petgraph

```rust
use petgraph::graph::{DiGraph, NodeIndex};
use std::collections::HashMap;

/// 从角色卡 JSON 构建 petgraph 有向关系图
pub fn build_relationship_graph(characters: &[CharacterCard]) -> DiGraph<CharacterNode, RelationshipEdge> {
    let mut graph = DiGraph::new();
    let mut node_indices: HashMap<String, NodeIndex> = HashMap::new();

    // Step 1: 每个角色一个节点
    for char in characters {
        let idx = graph.add_node(CharacterNode {
            id: char.base.id.clone(),
            name: char.base.name.clone(),
            importance: char.base.importance,
        });
        node_indices.insert(char.base.id.clone(), idx);
    }

    // Step 2: 从 initialRelationships 添加边（有向）
    for char in characters {
        let source = node_indices[&char.base.id];
        for (target_id, rel) in &char.base.initial_relationships {
            if let Some(&target) = node_indices.get(target_id) {
                graph.add_edge(source, target, RelationshipEdge {
                    label: rel.label.clone(),
                    trust: rel.trust,           // -10 ~ 10
                    familiarity: rel.familiarity, // 0 ~ 10
                    sentiment: rel.sentiment,    // -10 ~ 10
                    asymmetry: true,            // 标记为手动设置的非对称值
                });
            }
        }
    }

    // Step 3: 对没有反向边的边，自动推断反向值
    for edge in graph.edge_indices().collect::<Vec<_>>() {
        let (source, target) = graph.edge_endpoints(edge).unwrap();
        let has_reverse = graph.find_edge(target, source).is_some();
        if !has_reverse {
            let weight = graph.edge_weight(edge).unwrap().clone();
            // 反向推断：trust = trust * 0.7（人倾向于认为别人对自己没那么好）
            graph.add_edge(target, source, RelationshipEdge {
                trust: (weight.trust as f64 * 0.7) as i8,
                familiarity: weight.familiarity,
                sentiment: (weight.sentiment as f64 * 0.7) as i8,
                asymmetry: false,
            });
        }
    }

    graph
}
```

### 2.2 连通性验证与桥接

```rust
fn validate_and_bridge(graph: &mut DiGraph<CharacterNode, RelationshipEdge>, characters: &[CharacterCard]) {
    // 找到孤立节点（度数为 0 的角色）
    let isolated: Vec<_> = graph.node_indices()
        .filter(|n| graph.neighbors(*n).count() == 0 && graph.neighbors_directed(*n, Incoming).count() == 0)
        .collect();

    for node in isolated {
        let char = &characters[node.index()];
        // 找到与它最"可能"有关的角色（同阵营、同背景、同位置）
        if let Some(bridge_target) = find_closest_relation(char, characters, graph) {
            graph.add_edge(node, bridge_target, RelationshipEdge {
                label: "关联".into(),
                trust: 1,
                familiarity: 1,
                sentiment: 0,
                asymmetry: false,
            });
        }
    }
}
```

### 2.3 小世界属性验证

初始化后的关系图应满足：
- 任意两个角色之间路径长度 ≤ 2（小世界特性）
- 聚类系数 ≥ 0.4（初始群体高度内聚）
- 至少有一条"弱连接"（trust < 3 的边）作为未来剧情桥接

## 四、初始环境播种

### 4.1 资源分布原则：不实现自给自足

初始资源分布确保**没有任何一个地点能独立维持超过 3 天**：

| 地点 | 食物(kg) | 燃料(L) | 木材(kg) | 药品(unit) | 可维持天数 |
|------|---------|---------|---------|-----------|-----------|
| 五金店 | 25 | 15 | 40 | 5 | ~2 天 |
| 修车铺 | 0 | 80 | 10 | 0 | ~0 天 |
| 超市 | 80 | 0 | 0 | 15 | ~2 天 |
| 卫生院 | 0 | 0 | 0 | 40 | ~0 天 |

**这种分布强制**：角色必须在据点间移动、交易、协作。

### 4.2 资源方差播种

每次启动时资源量在 ±10% 范围内随机波动：

```rust
fn seed_resources(locations: &mut [LocationNode], rng: &mut ChaChaRng) {
    for location in locations {
        for resource in location.resources.values_mut() {
            let variance = 1.0 + rng.gen_range(-0.1..0.1);
            resource.current = (resource.current as f64 * variance) as u32;
            resource.current = resource.current.max(1);  // 至少留 1
        }
    }
}
```

### 4.3 初始环境状态

## 五、初始种子事件

### 5.1 三层危机种子

```rust
fn seed_crises(config: &PlotConfig, timeline: &Timeline) -> Vec<ScheduledEvent> {
    let mut events = Vec::new();

    // 即时危机：资源紧张，1-3 天内需要行动
    events.push(ScheduledEvent::at_tick(3, EventType::ResourceAlert {
        location: "hardware-store".into(),
        resource: "fuel".into(),
        threshold: 5.0,
        message: "五金店的柴油仅够维持不到三天，需要尽快补充",
    }));

    // 中期危机：重要角色登场，7-14 天
    events.push(ScheduledEvent::at_tick(
        timeline.ticks_from_days(7),
        EventType::CharacterArrival {
            character: "linshuang".into(),
            location: "clinic".into(),
            description: "张远在废弃药店遇到独自搜索物资的陌生女人",
        },
    ));

    // 远期危机：世界观扩展，20+ 天
    events.push(ScheduledEvent::at_tick(
        timeline.ticks_from_days(21),
        EventType::PlotAdvance {
            description: "有人在镇子边缘目击了疑似丧尸的生物，恐慌开始蔓延",
        },
    ));

    events
}
```

### 5.2 Hook 种子

除了预定的剧情事件，系统还播种一批"可能发生"的 hook——它们不一定会触发，但为早期世界提供张力：

```rust
fn seed_hooks(world: &WorldState) -> Vec<Hook> {
    vec![
        // 低温 → 冻伤风险 hook（条件触发）
        Hook::conditional(
            Condition::TemperatureBelow(-10),
            EventType::FrostbiteRisk,
            priority: HookPriority::Normal,
        ),
        // 物资减少 → 内部冲突 hook（概率触发）
        Hook::probabilistic(
            0.3,  // 每 tick 30%
            Condition::ResourceBelow("food", 20),
            EventType::InternalConflict,
            priority: HookPriority::Normal,
        ),
    ]
}
```

## 六、Opening Prompt

### 6.1 设计原则

Opening Prompt 不是"解释世界"，而是**制造疑问**：

| 原则 | 好的做法 | 差的做法 |
|------|---------|---------|
| 建立锚点 | "十一月的南方，下午四点半" | "故事发生在2025年..." |
| 反常日常 | "修电动车，水龙头冻住了" | "温度异常下降是由于北极涡旋..." |
| 人类连接 | 陈磊发微信"搞了条狗肉" | 角色独白介绍自己背景 |
| 系统不解释 | 触碰电台 → 脑子中出现信息 | "您的生存系统已激活"大段说明 |
| 留白 | "好像不是错觉" | "这是一个预兆，意味着末日即将来临" |

### 6.2 Prompt 模板

```rust
fn build_opening_prompt(world: &WorldState) -> Vec<Message> {
    let anchor_char = &world.characters[0];  // 张远
    let anchor_location = world.get_location(anchor_char.location.unwrap());
    let primary_relationship = world.relationship_graph
        .get_first_edge(anchor_char.id);

    vec![
        Message::system(SYSTEM_PROMPT_OPENING),
        Message::user(format!(
            "## 定场基本信息\n\
             时间：{}，{}，{}\n\
             地点：{}（{}）\n\
             温度：{}°C，{}\n\
             在场角色：{}\n\
             最近人际：{}（信任值：{}/10）\n\n\
             ## 当前异常\n\
             {}",  // 第一句话从反常日常切入
            world.timeline.now_date(),
            world.timeline.now_day_of_week(),
            world.timeline.now_time_of_day(),
            anchor_location.name,
            anchor_location.description_first_line(),
            world.environment.temperature,
            world.environment.weather_description(),
            anchor_char.name,
            primary_relationship.target_name,
            primary_relationship.trust,
            "气温在十二月初就降到了零下，这不对。但人们还没意识到这是末日。",
        )),
    ]
}
```

### 6.3 输出格式

Opening Prompt 的输出不是推演结果，而是**世界基调设定**：


Opening Prompt **可选执行**——如果用户从检查点恢复模拟，跳过此步。

## 七、验证与预热

### 7.1 启动验证清单

```rust
fn validate_world_initialization(world: &WorldState) -> Vec<InitWarning> {
    let mut warnings = Vec::new();

    // 1. 关系图连通性
    if !is_connected(&world.relationship_graph) {
        warnings.push(InitWarning::IsolatedCharacters);
    }

    // 2. 每个角色有位置
    for char in &world.characters {
        if char.location.is_none() && !char.flags.contains("unspawned") {
            warnings.push(InitWarning::MissingLocation(char.id.clone()));
        }
    }

    // 3. 资源可维持天数
    for location in &world.locations {
        let days = location.survival_days();
        if days < 1.0 {
            warnings.push(InitWarning::CriticalResource(location.id.clone()));
        }
    }

    // 4. 事件队列非空
    if world.event_queue.is_empty() {
        warnings.push(InitWarning::NoEventsScheduled);
    }

    warnings
}
```

### 7.2 预热 tick

启动后自动运行 3-5 个 FastForward tick（零 LLM，用户不可见），让数值系统稳定：

```rust
fn warmup_ticks(engine: &mut SimulationEngine, count: u64) {
    engine.state = EngineState::Running { speed: SimSpeed::FastForward };
    for _ in 0..count {
        engine.fastforward_tick();
    }
    engine.state = EngineState::Paused;
}
```

预热后，气温小幅下降、消耗了少量资源、角色基础状态初始化完成。用户看到的是一个已经"运转了一小会儿"的世界。

## 第一部分：超丰富角色卡体系

### 一、设计理念

角色卡分为**五层**，从不可变的 DNA 层到每 tick 变化的瞬时层：

```
层级 1: 元数据 DNA（schemaVersion, id, 创造日期）        ← 永不改变
层级 2: 静态基底（人格、背景、外貌、原型）                ← 作者/LLM 设定后不自动变
层级 3: 动态基底（技能树、关系网、知识、信念）            ← 随事件缓慢演化
层级 4: 生理心理状态（HP、情绪、压力、记忆流）            ← 每 tick 更新
层级 5: 瞬时状态（当前位置、当前行为、注意力焦点）        ← 随时变化
```

## 第二部分：世界数据热插拔系统

### 一、设计原则

1. **引擎不停机**—所有世界数据的增删改在运行时执行，不要求暂停
2. **自动重平衡**—插入新数据后引擎自动调整相关状态，不产生孤立元素
3. **API 驱动**—所有操作通过 REST API，为 Dashboard 准备
4. **全量校验**—每次修改后运行校验器，防止数据不一致

**角色中途插入流程：**

```rust
/// 角色插入引擎的全流程
async fn insert_character_mid_simulation(
    engine: &mut SimulationEngine,
    card: CharacterCardV3,
    location_id: Option<String>,    // 初始位置（None = 引擎自动分配）
    relationship_targets: Vec<(String, RelationshipDef)>,  // 初始关系
    arrival_event: bool             // 是否生成"登场"事件
) -> Result<InsertResult> {
    // Step 1: 验证角色卡完整性
    let errors = validate_character_card(&card);
    if !errors.is_empty() { return Err(ValidationError(errors)); }

    // Step 2: 将角色加入 ECS World
    let entity_id = engine.world.character_world.spawn((
        card.clone(),
        CharacterState::default(),
        EmotionState::from_personality(&card.personality.ocean),
    ));

    // Step 3: 更新 petgraph 关系图
    let node_idx = engine.world.relationship_graph.add_node(entity_id.into());
    for (target_id, rel) in &relationship_targets {
        if let Some(target_node) = engine.world.character_index.get(target_id) {
            engine.world.relationship_graph.add_edge(node_idx, *target_node, rel);
        }
    }

    // Step 4: 分配初始位置
    if let Some(loc_id) = location_id {
        engine.world.assign_character_location(entity_id, &loc_id)?;
    } else {
        // 自动分配：找到最近的有资源余量的位置
        let loc = engine.world.find_nearest_shelter(&card);
        engine.world.assign_character_location(entity_id, &loc)?;
    }

    // Step 5: 更新标签索引
    engine.world.tag_index.ingest_character(&card);

    // Step 6: 可选—生成"登场"事件
    if arrival_event {
        engine.world.event_queue.push_back(ScheduledEvent {
            trigger_time: engine.world.timeline.now + Duration::minutes(5),
            event_type: EventType::CharacterArrival(entity_id),
            participants: vec![entity_id],
            priority: EventPriority::Normal,
        });
    }

    // Step 7: 重计算全局统计
    engine.stats.total_characters = engine.world.character_world.len() as u64;

    Ok(InsertResult { entity_id, node_idx })
}
```

**地点中途插入流程：**

```rust
async fn insert_location_mid_simulation(
    engine: &mut SimulationEngine,
    location: LocationNode,
    connections: Vec<(String, LocationEdge)>,  // 连接到现有地点
) -> Result<InsertResult> {
    // Step 1: 校验
    validate_location(&location)?;

    // Step 2: 加入地点图
    let node_idx = engine.world.location_graph.add_node(location);

    // Step 3: 建立连接边
    for (target_id, edge) in &connections {
        if let Some(target_idx) = engine.world.location_index.get(target_id) {
            engine.world.location_graph.add_edge(node_idx, *target_idx, edge.clone());
        }
    }

    // Step 4: 自动重分配——如果新地点有资源，通知附近角色
    let notification = engine.world.announce_new_location(node_idx);
    engine.ws_broadcaster.send(EngineEvent::Log {
        level: LogLevel::Info,
        message: format!("新地点被发现：{}", location.name),
    });

    Ok(InsertResult { node_idx })
}
```

### 三、热插拔校验器

每次插入/修改后自动运行的校验规则：

```rust
pub async fn validate_world_consistency(world: &WorldState) -> Vec<ValidationIssue> {
    let mut issues = Vec::new();

    // 1. 角色校验
    for char in &world.characters {
        // 1a. 每个角色有位置
        if char.location.is_none() && !char.flags.contains("unspawned") {
            issues.push(issue(Level::Error, "角色无位置", &char.id));
        }
        // 1b. 位置存在于地点图中
        if let Some(loc) = &char.location {
            if !world.location_index.contains_key(loc) {
                issues.push(issue(Level::Critical, "角色在不存在的地点", &char.id));
            }
        }
        // 1c. 关系图的边对应的角色都存在
        for rel in char.active_relationships() {
            if !world.character_index.contains_key(&rel.target) {
                issues.push(issue(Level::Critical, "关系指向不存在的角色", &char.id));
            }
        }
    }

    // 2. 地点校验
    for node in world.location_graph.node_indices() {
        let loc = &world.location_graph[node];
        // 2a. 资源非负
        for (_, res) in &loc.resources {
            if res.current < 0 {
                issues.push(issue(Level::Error, "负资源", &loc.id));
            }
        }
        // 2b. 连接边指向存在的地点
    }

    // 3. 事件校验
    for event in &world.event_queue {
        // 3a. 事件参与者都存在
        for p in &event.participants {
            if !world.character_index.contains_key(p) {
                issues.push(issue(Level::Warning, "事件参与不存在的角色", &event.id));
            }
        }
    }

    // 4. 阵营校验
    for faction in &world.factions {
        // 4a. 成员都存在
        for m in &faction.members {
            if !world.character_index.contains_key(m) {
                issues.push(issue(Level::Error, "阵营成员不存在", &faction.id));
            }
        }
    }

    issues
}
```
